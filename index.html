LAMP LANP

  • LAMP linux+apache+mysql+php python nginx

1.nginx安装 yum -y install nginx

2.mysql安装 yum -y install mysql mysql-server mysql-devel

  • 创建用户授权创建表 我在MySQL里面做了介绍，就不在这里阐述了

3.php安装 yum -y install php

4.php-mysql安装 yum -y install php-mysql [安装了什么编程语言，就要安装相对应的（语言-mysql）包 比如：php php-mysql]
5.dns安装 yum -y install bind bind-libs

6.spawn-fcgi安装 需要下载安装包
配置spawn-fcgi

vim /etc/sysconfig/spawn-fcgi
取消下面两行代码的注释，如果第二行代码是-u apache -g apache的话，需要改成-u nginx -g nginx
SOCKET=/var/run/php-fcgi.sock
OPTIONS=”-u nginx -g nginx -s $SOCKET -S -M 0600 -C 32 -F 1 -P /var/run/spawn-fcgi.pid – /usr/bin/php-cgi”
配置nginx 以虚拟主机的方式
vim /etc/nginx/conf.d/www.conf
在配置虚拟主机的基础下加入以下代码
location ~ .php$ {
root /var/www/html/www;
fastcgi_pass unix:/var/run/php-fcgi.sock;
fastcgi_index index.php;
fastcgi_param SCRIPT_FILENAME $document_root$fastcgi_script_name;
include fastcgi_params;
}

nagios

  • 在工作中，尤其是做运维，对服务器的管控，很重要，今天我搭建了服务器监控nagios。工作原理：数据收集–>数据存储–>数据分析–>报警处理–>数据显示

  • 需求：通过nagios对磁盘和web服务器进行监控并且发邮件报警

  • 需求监控30.37的 30.37主机//http:80 端口 OK 5m KO 1m 3次状态一样最终确定服务器

  • 30.37 7*24 监控

  • service 06-24

  • warn | fail | OK unknown ：

  • email ： proo@qq.com

  • notify in 2h 发送 报警的频率

  • time 5*8 上班时间

需要安装软件 nagios nagios-plugins nagios-web

  • 我是在官网找到的rpm包 ，然后通过记录到yum然后配置软件仓库。

    yum install nagios nagios-plugins nagios-web

创建用户

useradd nagios
groupadd nagcmd   
gpasswd -a nagios nagcmd     ---nagios添加到nagcmd组

修改配置文件

  • 添加主机配置文件

    vim /etc/nagios/objects/hosts/30.6.cfg
    define host{

    use                     linux-server
host_name               vm130
alias                   vm130
address                 192.168.32.130
max_check_attempts      3

    }

    define service{

    use                             local-service
host_name                       vm130
service_description             HTTP
check_command                   check_http
notifications_enabled           1

    }

define contact{

linux+hexo

  • 做为一个计算机在运维道路上的人应该有一个自己的博客，我找到了很多开源项目，因为我是学生党，没有钱只能找免费的项目做博客，这里我找到了linux+hexo+github 完全托管搭建自己的博客。

1.首先去github官网去申请自己的账号。

  • github 为用户提供了用户名的二级博客，同时拥有300M的免费空间，写博客完全够用了。

注意：在部署环境的时候需要linux镜像的软件仓库，以及epel项目的仓库。

  • 这里软件yum仓库我就不做介绍了

安装基本的软件，准备环境。

yum -y install gcc zlib-devel openssl-devel perl cpio expat-devel gettext-devel curl autoconf

安装git node

yum istall git nodejs

python必须是2.6以上的版本

安装npm

yum install npm

安装hexo

npm install -g hexo

创建文件blog目录

  • 创建一个blog的工作目录

    mkdir /hexo
    cd /hexo
    hexo init #初始化hexo
    ls # 出现很多文件 即表示成功
    npm install gexo-deployer -git –save #安装依赖

生成静态网页

hexo generate #可以用g表示generate
  • 生成完毕之后，我们可以看到多了一个public文件夹，这就是我们所谓的静态博客的目录，如果我们需要部署到服务器或者托管平台，只要将hexo生成静态之后，将public文件夹里的文件传上去就可以了,其他系统文件还是放在本地。

本地预览

hexo server
  • 浏览器中输入http://localhost:4000 可以在本地查看

配置_config.yml

deploy:
  type:git
  repository: git@github.com:prokao/prokao.github.io.git
  branch:master
